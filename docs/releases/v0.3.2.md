## inD3X Art v0.3.2

**Quality & reliability release** — accessibility, CI gates, editor performance, and desktop window controls.

### Downloads

| Platform | Artifact |
|----------|----------|
| **Windows** | `inD3X Art_*_x64-setup.exe` (NSIS installer) |
| **Linux** | `inD3X Art_*_amd64.AppImage` (portable) |

SHA256 checksums are attached as `SHA256SUMS` on this release.

### Highlights

- **Release CI gate** — every `v*` tag runs typecheck, lint, unit tests, Clippy, Rust tests, and integration E2E *before* Tauri bundles are built.
- **Window maximize/restore** — custom title bar controls now work on Windows (Tauri permissions + drag-region fixes).
- **Editor performance** — singleton pixel worker, sparse Comlink transfers, layer pixel cache.
- **Accessibility** — command palette combobox ARIA, assertive error toasts, session-restore dialog semantics.
- **E2E fault injection** — targeted `failOps` works; save failure tests use `saveBatch` path.

### Added

- Singleton pixel worker (`pixelWorkerClient.ts`), sparse worker protocol, layer pixel cache
- Save path validation, explorer source basename labels, expanded golden/unit tests
- `rust-toolchain.toml`, fixture `pack.mcmeta` files, mymod `custom_block.png`

### Fixed

- Stale explorer selection after reindex; comparator 3D without before-model
- Multipart `when` JSON booleans; mock IPC blocked in production builds
- Session-restore E2E dialog title; catalog/lifecycle test deadlocks
- Panel error boundary reset on project change; context menu portal + clamp

### Upgrade notes

No migration required. Settings and project caches are compatible with v0.3.1.
